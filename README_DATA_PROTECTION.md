# Data Protection Dashboard

Раздел предназначен для переноса регламентной Word-таблицы по резервированию информационных систем в веб-интерфейс.

## Основная идея

Это не журнал фактически выполненных backup-задач, а план/политика резервирования:

- какая информационная система защищается;
- какая информация должна резервироваться;
- какой максимальный объём данных;
- по какой процедуре выполняется backup;
- какой срок хранения;
- кто ответственный.

## Разделы интерфейса

`Data Protection` показывает:

- KPI по количеству систем и суммарному объёму;
- количество политик с Full и Incremental;
- количество записей без ответственного;
- группировку по типам backup;
- таблицу политик резервирования.

## Группы

Поддерживаются группы:

- Filesystem;
- Internal Database;
- MS SQL Server;
- Virtual Environment.

## Рекомендуемый сценарий переноса из Word

1. Открыть Word-документ.
2. Скопировать таблицу в Excel.
3. Добавить колонку `Тип backup`, если она визуально была заголовком группы.
4. Сохранить `.xlsx`.
5. В UI открыть `Data Protection` → `Импорт Excel`.

## Связь с фактическим мониторингом

В дальнейшем этот раздел можно связать с `/backup-monitor`:

- Data Protection = что должно резервироваться;
- Backup Monitor = что реально резервируется и когда последний раз прошло успешно.

На следующем этапе можно сделать сверку:

- есть в плане, но нет фактического backup-объекта;
- по плану backup ежедневный, но фактически последний backup старше 24 часов;
- нет ответственного;
- превышен объём.
